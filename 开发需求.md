## 6.8 PDRS Step

实现本地评分引擎。

公式：

```text
PDRS =
25% × Opportunity Score
+ 25% × Competitive Score
+ 25% × Specification Score
+ 15% × Prototype Score
+ 10% × Prompt Readiness Score
```

决策规则：

```text
PDRS >= 85: export
70 <= PDRS < 85: export_with_risk
50 <= PDRS < 70: revise_before_export
PDRS < 50: abandon_or_reframe
```

评分维度解释必须展示在 UI 上。

## 评分初版规则

Opportunity Score：
根据 idea 完整度、target user 是否明确、industry 是否明确、MVP 可实现性估算。

Competitive Score：
根据 competitors 数量、相关性、threatLevel、reuseStrategy 和 differentiationScore 计算。

Specification Score：
根据 PRD 是否包含目标用户、痛点、用户故事、验收标准、数据模型、API 合同计算。

Prototype Score：
本版本暂用 flow/prototype readiness 替代：
是否有页面结构、核心流程、非目标范围、关键任务路径。

Prompt Readiness Score：
根据 Codex Pack 是否包含 goal、stack、pages、data models、API、tasks、acceptance criteria、run commands 计算。

必须确保评分可解释：每个子分数都要有 reasons。
